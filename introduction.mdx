`changed-files` is a GitHub Action that identifies exactly which files changed between two commits and exposes them as structured workflow outputs. Instead of running every CI step on every push, you can scope work to only the files that actually changed — making workflows faster, cheaper, and easier to reason about.

## What it does

`changed-files` compares two commits and produces a categorized list of every file that changed between them. On `pull_request` events it compares the PR branch against the target branch. On `push` events it compares the current commit against the preceding remote commit. The outputs include per-category file lists (e.g. `added_files`, `deleted_files`), counts for each category, and boolean flags like `any_changed` that make conditional step execution straightforward.

The action does not detect uncommitted or staged changes made during a workflow run. It only reports changes that exist in committed git history.

## Supported events

`changed-files` works with a wide range of GitHub Actions trigger events:

- `pull_request` and `pull_request_target` — compares the PR branch against the base branch
- `push` — compares the current commit against the preceding remote commit
- `merge_group` — supports merge queue workflows
- `release` — detects changes relative to the previous release tag
- `schedule`, `workflow_dispatch`, `workflow_run`, `issue_comment`, and more

<Note>
  GitHub's REST API is only available for `pull_request` events. For `push` and all other events, the action uses the local `.git` directory, which requires `actions/checkout` to be present in your workflow.
</Note>

## Supported platforms

The action runs on all GitHub Actions runner operating systems:

- **Ubuntu** (recommended for most workflows)
- **macOS**
- **Windows** (use `use_posix_path_separator: true` for consistent path formatting)

It supports **GitHub-hosted runners**, **self-hosted runners**, and **GitHub Enterprise Server**.

## File change type acronyms

Each file in a changed-files output is classified by Git's diff status letter. These acronyms appear throughout this documentation and in output names like `added_files` or `type_changed_files`.

| Acronym | Meaning      |
|:-------:|:-------------|
| A       | Added        |
| C       | Copied       |
| M       | Modified     |
| D       | Deleted      |
| R       | Renamed      |
| T       | Type changed |
| U       | Unmerged     |
| X       | Unknown      |

The `all_changed_files` output combines ACMR (all except deleted, type-changed, unmerged, and unknown). The `all_modified_files` output combines ACMRD. The `all_changed_and_modified_files` output combines all categories (ACMRDTUX).
