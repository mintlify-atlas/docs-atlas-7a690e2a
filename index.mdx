`changed-files` is a GitHub Action that detects exactly which files changed in a pull request or push event — so your workflows only run the steps that matter. It supports glob patterns, YAML-based file grouping, directory outputs, matrix job generation, and file recovery, all in 0–10 seconds.

## How it works

`changed-files` compares two commits and produces a categorized list of every file that changed between them. On pull request events it compares the PR branch against the base branch. On push events it compares the current commit against the preceding remote commit.

<Steps>
  <Step title="Add actions/checkout to your workflow">
    Check out your repository so the action has access to the local git history.

    ```yaml
    - uses: actions/checkout@v4
    ```

    <Note>For push events, set `fetch-depth: 0` or `fetch-depth: 2` to ensure at least two commits are available for comparison.</Note>
  </Step>
  <Step title="Add the changed-files step">
    Run the action and capture its output with an `id`.

    ```yaml
    - name: Get changed files
      id: changed-files
      uses: tj-actions/changed-files@v47
    ```
  </Step>
  <Step title="Use the outputs in subsequent steps">
    Reference any output using `steps.<id>.outputs.<output_name>`.

    ```yaml
    - name: List all changed files
      env:
        ALL_CHANGED_FILES: ${{ steps.changed-files.outputs.all_changed_files }}
      run: |
        for file in ${ALL_CHANGED_FILES}; do
          echo "$file was changed"
        done
    ```
  </Step>
</Steps>
